# Архитектура системы Manus (Sandbox Runtime)
> Детальный разбор для построения собственного ИИ-агента

Архитектура Manus в директории `/opt/.manus/` построена по принципу изолированной песочницы (sandbox) с глубокой интеграцией браузерной автоматизации, виртуального дисплея и проксирования API. 

Ниже представлена полная схема и описание компонентов, которые вам понадобятся для создания аналогичной плавной и мощной системы.

---

## 1. Общая структура директорий

Система разделена на 4 ключевых модуля:

| Директория | Назначение |
|---|---|
| `/opt/.manus/current/` | Основной движок агента (Runtime). Содержит бинарник сервера, JS-скрипты для инъекций в браузер и API-клиент. |
| `/opt/.manus/neko/` | Виртуальный дисплей и стриминг (на базе n.eko). Позволяет агенту "видеть" экран и взаимодействовать с ним, а пользователю — наблюдать за процессом. |
| `/opt/.manus/deploy/` | Шаблоны для быстрого развертывания веб-приложений (Flask, React, Next.js и др.). |
| `/opt/.manus/webdev/` | Инструменты для веб-разработки внутри песочницы. |

---

## 2. Ключевые компоненты для вашего ИИ-агента

Чтобы построить такую же систему, вам нужно реализовать следующие слои:

### Слой 1: Виртуальный дисплей и стриминг (Neko)
Manus использует модифицированную версию [n.eko](https://neko.m1k1o.net/) — это самодостаточный браузер в Docker/виртуальной машине, который стримит экран через WebRTC.
*   **Файлы:** `/opt/.manus/neko/config/neko.yml`, `ice_servers.json`
*   **Как повторить:** Установите Xvfb (виртуальный X-сервер) + PulseAudio + n.eko. Это даст вашему агенту реальный рабочий стол, где он сможет двигать мышкой и кликать, а не просто парсить HTML.

### Слой 2: Глубокая браузерная автоматизация (JS Injections)
Вместо того чтобы полагаться только на Playwright/Selenium, Manus использует набор кастомных JS-скриптов (`/opt/.manus/current/app_data/js/`), которые инжектируются в страницу для анализа DOM:
*   `collectClickableElements.js` — находит все кликабельные элементы, вычисляет их координаты и присваивает им уникальные ID.
*   `analyzeHtmlElements.js` — строит упрощенное дерево элементов, отбрасывая невидимые и неважные теги.
*   `checkElementsBelowHeight.js` — умный скроллинг, проверяет есть ли контент ниже текущего экрана.
*   **Как повторить:** Напишите JS-скрипты, которые будут собирать координаты (x, y, width, height) всех кнопок и ссылок, и передавайте этот JSON вашему Python-агенту. Агент будет принимать решение и отправлять команду "кликнуть по координатам X, Y".

### Слой 3: Единый API Прокси (`data_api.py`)
Агент не хранит десятки API-ключей от разных сервисов. Вместо этого используется единый прокси-клиент.
*   **Файл:** `/opt/.manus/current/data_api.py`
*   **Как повторить:** Создайте микросервис (например, на FastAPI), который будет принимать запросы от агента (с единым внутренним токеном) и перенаправлять их к OpenAI, GitHub, Google Drive и т.д., подставляя нужные ключи на стороне сервера.

### Слой 4: Шаблонизатор проектов (`deploy/templates/`)
Когда Manus просят "создать сайт", он не пишет его с нуля. Он берет готовый шаблон и модифицирует его.
*   **Пример:** `/opt/.manus/deploy/templates/flask/` содержит готовый `main.py`, `models/user.py`, `routes/user.py` с настроенной SQLAlchemy.
*   **Как повторить:** Создайте папку с базовыми шаблонами (React, Flask, FastAPI). Когда пользователь просит приложение, агент копирует шаблон и с помощью LLM дописывает только нужную бизнес-логику.

## 3. Схема взаимодействия (Как это работает плавно)

1.  **Инициализация:** Запускается Xvfb (виртуальный экран) и Neko (WebRTC стриминг).
2.  **Агент (Python):** Запускается основной цикл (Agent Loop).
3.  **Браузинг:** Агент открывает Chromium через Playwright.
4.  **Анализ:** В страницу инжектируется `collectClickableElements.js`. Агент получает JSON со всеми кнопками и их координатами.
5.  **Действие:** LLM решает, куда кликнуть. Playwright выполняет клик по координатам.
6.  **Кодинг:** Если нужно написать код, агент берет шаблон из `/deploy/templates/`, модифицирует его и запускает локально.

## 4. Рекомендации по сборке

Для создания аналогичной системы вам потребуются следующие технологии:
*   **Язык:** Python 3.11+
*   **Браузер:** Playwright + кастомные JS-инъекции
*   **Виртуализация экрана:** Xvfb + n.eko (WebRTC)
*   **LLM Оркестрация:** LangChain или чистый OpenAI SDK с поддержкой Function Calling (Tools)
*   **Изоляция:** Docker или системные контейнеры (LXC) для каждого агента.
