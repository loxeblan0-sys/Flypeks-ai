# Комплексный анализ проекта ai-manus

## Введение

Проект **ai-manus** представляет собой универсальную платформу для создания и управления AI-агентами, спроектированную для полной изоляции и развертывания в частной инфраструктуре. Ключевой особенностью системы является её способность выполнять задачи в изолированной песочнице (sandbox), используя разнообразные инструменты, такие как веб-браузер, терминал и файловая система. Это позволяет агентам безопасно взаимодействовать с внешней средой, производить поиск информации, работать с кодом и выполнять сложные многоэтапные задачи. Проект является открытым исходным кодом, активно развивается и стремится предоставить гибкую и расширяемую основу для создания специализированных корпоративных Manus-приложений [1].

## Архитектура и Дизайн

Архитектура `ai-manus` построена на микросервисном подходе, что обеспечивает гибкость, масштабируемость и простоту обслуживания. Проект состоит из трёх основных, независимых друг от друга компонентов: `frontend`, `backend` и `sandbox`. Взаимодействие между ними осуществляется через REST API и WebSocket для обмена данными в реальном времени [1].

<br>

| Компонент | Технологии | Описание |
| :--- | :--- | :--- |
| **Frontend** | Vue.js, TypeScript, Vite, TailwindCSS | Пользовательский интерфейс для взаимодействия с агентом, просмотра его действий и результатов. |
| **Backend** | Python, FastAPI, MongoDB, Redis | Основной сервер, управляющий логикой агентов, сессиями, аутентификацией и оркестрацией задач. |
| **Sandbox** | Docker, Ubuntu, Python, Supervisord | Изолированная среда выполнения для каждого задания, содержащая необходимые инструменты (браузер, терминал). |

<br>

Процесс взаимодействия компонентов выглядит следующим образом:
1.  **Инициация сессии:** Пользователь через `frontend` отправляет запрос на создание новой сессии. `Backend` получает запрос и создает новый экземпляр `sandbox` с помощью Docker API, монтируя сокет `/var/run/docker.sock` для управления контейнерами.
2.  **Выполнение задачи:** Пользователь отправляет сообщение (задачу) в рамках созданной сессии. `Backend` передает это сообщение агенту (Plan-Act Agent), который начинает процесс планирования и выполнения.
3.  **Использование инструментов:** В ходе выполнения задачи агент может использовать инструменты, доступные в `sandbox`. Взаимодействие с инструментами происходит через API, предоставляемое `sandbox`.
4.  **Обратная связь:** Все события, генерируемые в процессе работы агента (создание плана, вызов инструментов, вывод в консоль), отправляются с `backend` на `frontend` в реальном времени с использованием технологии Server-Sent Events (SSE).

Такая архитектура позволяет полностью изолировать выполнение задач, обеспечивая безопасность и предотвращая влияние одной задачи на другую. Использование Docker для `sandbox` упрощает управление зависимостями и обеспечивает консистентность среды выполнения [1].

## Backend

Бэкенд является ядром системы `ai-manus`, реализован на Python с использованием асинхронного веб-фреймворка FastAPI. Он отвечает за всю основную логику работы агентов, управление сессиями, аутентификацию пользователей и взаимодействие с другими компонентами системы. Архитектура бэкенда следует принципам чистой архитектуры, разделяя логику на слои домена, приложения и инфраструктуры.

### Ключевые компоненты и технологии

-   **FastAPI:** Обеспечивает высокую производительность и асинхронную обработку запросов, что критически важно для системы, работающей в реальном времени.
-   **MongoDB:** Используется в качестве основной базы данных для хранения информации о сессиях, агентах, пользователях и истории событий. Документо-ориентированная природа MongoDB хорошо подходит для хранения сложных и вложенных структур данных, таких как планы и события агента.
-   **Redis:** Применяется для двух основных целей: как кэш для часто запрашиваемых данных и как брокер сообщений для организации асинхронных задач через Redis Streams. Это позволяет отделить выполнение долгосрочных задач агента от основного потока обработки запросов.
-   **Pydantic:** Широко используется для валидации данных и управления настройками, обеспечивая строгую типизацию и надежность при обмене данными между компонентами.

### Логика работы агента

Центральным элементом бэкенда является реализация агента, следующего модели **Plan-and-Act (PlanAct)**. Этот подход разделяет процесс выполнения задачи на два основных этапа: планирование и исполнение.

1.  **PlannerAgent:** На основе запроса пользователя и текущего контекста этот агент создает пошаговый план действий. План представляет собой список атомарных шагов, которые должен выполнить исполнитель.
2.  **ExecutionAgent:** Этот агент последовательно выполняет шаги, сгенерированные планировщиком. Для каждого шага он выбирает и вызывает соответствующий инструмент (например, `shell_exec` или `browser_navigate`).

Этот цикл планирования и выполнения повторяется, при этом план может динамически обновляться на основе результатов выполнения предыдущих шагов. Вся логика инкапсулирована в `PlanActFlow`, который координирует работу обоих агентов.

### Интеграция инструментов

Инструменты являются ключевым механизмом, с помощью которого агент взаимодействует с внешним миром. В `ai-manus` реализован гибкий механизм добавления и использования инструментов. Каждый инструмент (например, `ShellTool`, `BrowserTool`, `FileTool`) представляет собой класс с набором методов, декорированных как `@tool`. Эта декорация позволяет системе автоматически обнаруживать доступные инструменты и их параметры, генерируя схему для передачи в LLM. Агент-исполнитель, получив от LLM команду на вызов инструмента, делегирует выполнение соответствующему классу.

Проект также поддерживает интеграцию с внешними инструментами через **Model Context Protocol (MCP)**, что позволяет расширять возможности агента без модификации основного кода, подключая сторонние сервисы и API [2].

## Frontend

Пользовательский интерфейс `ai-manus` разработан как одностраничное приложение (SPA) с использованием **Vue.js 3**, **TypeScript** и **Vite**. Для стилизации применяется **TailwindCSS**, что позволяет быстро создавать современный и адаптивный дизайн. Фронтенд обеспечивает интуитивно понятное взаимодействие с агентом, позволяя пользователям не только отправлять задачи, но и в реальном времени наблюдать за ходом их выполнения.

### Основные функциональные возможности

-   **Интерактивный чат:** Основной интерфейс для общения с агентом, поддерживающий отправку текстовых сообщений и загрузку файлов.
-   **Визуализация работы агента:** В реальном времени отображаются все действия агента: созданный план, вызываемые инструменты, их параметры и результаты. Это обеспечивает полную прозрачность процесса.
-   **Просмотр и перехват управления:** Пользователь может просматривать содержимое инструментов, таких как браузер или терминал, в режиме реального времени. Более того, реализована функция "перехвата управления" (takeover), позволяющая пользователю временно приостановить работу агента и самостоятельно взаимодействовать с инструментом (например, для ввода капчи или прохождения аутентификации).
-   **Многоязычная поддержка:** Интерфейс поддерживает несколько языков, включая английский и китайский, что упрощает его использование для международной аудитории.
-   **Управление сессиями:** Пользователи могут создавать новые сессии (задачи), просматривать историю прошлых сессий и удалять их.

Для отображения удаленного браузера используется библиотека **noVNC**, которая позволяет транслировать VNC-сессию из `sandbox` через WebSocket прямо в браузер пользователя. Это создает иллюзию полного контроля над удаленным рабочим столом, хотя на самом деле взаимодействие происходит в изолированном Docker-контейнере [1].

## Sandbox

Песочница (`sandbox`) является ключевым элементом безопасности и изоляции в `ai-manus`. Каждая задача, запущенная пользователем, выполняется в отдельном Docker-контейнере, что гарантирует, что процессы одной задачи не могут повлиять на хост-систему или другие задачи. Этот подход также обеспечивает чистоту и консистентность окружения для каждого нового задания.

### Технологии и реализация

-   **Docker:** Основа для создания изолированных сред. `Backend` динамически создает и управляет жизненным циклом `sandbox`-контейнеров.
-   **Ubuntu:** В качестве базового образа для `sandbox` используется Ubuntu, что предоставляет стандартное и знакомое Linux-окружение.
-   **Supervisord:** Используется для управления процессами внутри контейнера. Он обеспечивает запуск и мониторинг всех необходимых сервисов, таких как API для инструментов и VNC-сервер.
-   **Python & FastAPI:** Внутри `sandbox` запущен легковесный FastAPI-сервер, который предоставляет REST API для взаимодействия с инструментами (терминал, файловая система).
-   **Xvfb, x11vnc, websockify:** Этот стек технологий используется для обеспечения возможности просмотра и управления графическим интерфейсом браузера. `Xvfb` создает виртуальный дисплей, `x11vnc` запускает VNC-сервер на этом дисплее, а `websockify` преобразует VNC-трафик в WebSocket для передачи на `frontend`.

Каждый `sandbox`-контейнер имеет ограниченное время жизни (TTL), по истечении которого он автоматически уничтожается, что предотвращает накопление неиспользуемых ресурсов. Такая эфемерная природа песочниц также повышает безопасность, так как любые изменения, внесенные в контейнер, теряются после его удаления [1].

## Заключение

Проект `ai-manus` представляет собой мощную и хорошо спроектированную платформу для создания универсальных AI-агентов. Его архитектура, основанная на микросервисах и полной изоляции задач в Docker-контейнерах, обеспечивает высокий уровень безопасности, масштабируемости и гибкости. Использование современных технологий, таких как FastAPI, Vue.js, MongoDB и Redis, делает систему производительной и удобной для разработки и поддержки.

Ключевыми преимуществами проекта являются:

-   **Полная приватность:** Возможность развертывания всей системы в собственной инфраструктуре без зависимости от внешних облачных сервисов.
-   **Безопасность:** Изоляция выполнения задач в `sandbox` предотвращает несанкционированный доступ и защищает хост-систему.
-   **Расширяемость:** Гибкая система инструментов и поддержка протокола MCP позволяют легко добавлять новые возможности и интегрироваться с другими сервисами.
-   **Прозрачность:** Пользовательский интерфейс обеспечивает полный контроль и наглядность процесса работы агента, включая возможность перехвата управления.

Проект активно развивается, о чем свидетельствует дорожная карта и регулярные обновления. Сообщество вокруг проекта также растет, что способствует его улучшению и появлению новых идей. `ai-manus` имеет большой потенциал стать стандартной платформой для создания корпоративных и специализированных AI-агентов, способных решать широкий круг задач в автоматическом режиме.
